About APXV

APXV (Attested Proof Execution Verified) is an air-gapped governed agent platform: markdown rules, signed capabilities, chained audit, Groth16 proofs, local API — bring your own LLMs. The open-source runtime is a foundation to build on, not a finished vertical product.

Teams are adding AI everywhere, but rarely adding governed execution — runtime policy, approval workflows, audit chains, and proofs you can verify without a cloud vendor.

  • Local control
  • Runtime governance
  • Verifiable attestation

Platform

The runtime: store, audit, capabilities, ZK, API. Full docs on GitHub.

  • Governed agents

    Living markdown rules, workflows, and knowledge at runtime.

  • Signed capabilities

    Per-agent permissions enforced before actions run.

  • Governance approval

    Propose → approve → apply for policy changes.

  • Redaction v3

    Format-aware pattern redaction with structured entities[] output.

  • Immutable artifacts

    SQLite index and content-addressable storage.

  • Chained audit logs

    Structured JSON events linked for verification.

  • Dual-track Groth16

    3 governance + entity proofs per attest (incl. merkle-inclusion, compliance) — independently verifiable.

  • Voice privacy

    STT → redact → attest with voice-redaction proof (simulated or local Vosk).

  • Ceremony transparency

    Tier A/B VK transcript + publishable verifier bundle (VKs only).

  • API v2

    /api/v2/* contract for Pack Studio, jobs SSE, governance, and uploads.

  • Sovereign bootstrap

    Operator-owned ZK keys on local storage — Docker, desktop, and native share one trust contract.

  • Desktop app

    Windows MSI and Linux deb/AppImage — bootstrap wizard, tray, local API.

  • Operator UI

    React + Tauri console — pipelines, artifacts, verify, Pack Studio.

Python 3.9+, Rust / arkworks, optional E2EE (--encrypt), stdlib HTTP API v1+v2, Docker, ui/ operator console

Agent packs

Packs are not the platform. They are vertical bundles on top — governance specs, install steps, and acceptance for a use case.

APXV core

  • Runtime, audit, store, ZK, local API
  • 3-agent pipeline pattern (redact → orchestrate → attest)
  • Signed capability framework
+

Agent pack

  • Rules, workflows, knowledge for a vertical
  • Install path + acceptance checklist
  • Binds core agents to a real use case
  1. 1
    Governance bundle

    Pack ships markdown specs — e.g. redaction rules, workflow steps, knowledge base.

  2. 2
    Apply to runtime

    Onboarding runs the reference pack out of the box. Production uses propose → approve → apply.

  3. 3
    Pipeline executes

    Agents read living specs at runtime, write immutable artifacts, record chained audit.

  4. 4
    Attest and verify

    Dual-track Groth16 proofs; anyone can verify with serialized proof + VK — no re-proving.

Shipped — v1.2.0

Not building: cloud SaaS, certification claims, bundled LLM, or “magic compliance.”

Scope

Honest about what ships today.

Includes

  • Local API on 127.0.0.1:8741 (v2 + legacy v1)
  • Sovereign bootstrap on desktop, Docker, and native paths
  • Desktop app (Windows + Linux) + operator UI + Pack Studio
  • Pipeline: redact → orchestrate → attest → verify
  • Three official agent packs + custom pack scaffold
  • Dual-track ZK (merkle-inclusion, compliance), optional E2EE, voice attest
  • BYO ML redaction hook (optional; audit only)
  • Ceremony transcript + verifier bundle export
  • Pluggable LLM backends (Ollama example)
  • Install scripts (native + Docker), doctor, integrity diagnostics

Does not include

  • HIPAA, SOC2, or GDPR certification
  • Enterprise DLP-grade redaction
  • Multi-party MPC trusted setup (single-party Groth16 today)
  • Protection against malicious insiders
  • Bundled LLM — you add your own
