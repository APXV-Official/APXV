# APXV1 — Groth16 Circuit Inventory (v1.2.2)

APXV1 uses **Groth16 over BN254** (arkworks 0.4) in two Rust crates. This document separates **circuits that exist** from **circuits proven on the default attest path**.

## Verification scope

Independent verification checks proof math and VK lineage. Setup honesty is a separate trust assumption — see [CEREMONY.md](CEREMONY.md).

---

## Governance track (`apx-circuits`) — 3 circuits

| Circuit | On `--attest`? | What it binds (summary) |
|---------|----------------|-------------------------|
| `redaction` | Yes | Original ≠ redacted hash; non-zero redaction count |
| `rule-binding` | Yes | Rule hash + redaction proof commitment |
| `pipeline` | Yes | Governance spec hashes + agent chain + decision |

All three are verified by `verify_attestation --real-zk`.

---

## Entity track (`apx-zk`) — 8 circuits in crate

| Circuit | On default `--attest`? | What it binds (summary) | Product feature required |
|---------|------------------------|-------------------------|--------------------------|
| `redaction-v1` | Yes | Merkle root, entity digest, document hashes, leaf commitments | Entity list in artifact (Agent 2 `entities[]`) |
| `core-redaction` | Yes | Merkle root, entity count, document hashes | Same |
| `batch-merkle` | Yes (1–4 entities) | Merkle root + inclusion paths for up to 4 leaves | Entity count ≤ 4 |
| `voice-redaction` | Yes (voice path) | Entity count, voice policy id, document hashes | `voice_session` from voice pipeline |
| `merkle-inclusion` | Yes (per entity, up to 8) | Single leaf inclusion in Merkle tree | One witness per entity index on default attest |
| `compliance` | Yes (when redaction occurred) | Entity count + compliance policy id (1–5) | Default policy id 1 (single-doc); pack may set 2–5 |
| `normalization` | No | Normalization feature bitmap, entropy, hash change | Pre-redaction normalization telemetry (v1.2+) |
| `threat` | No | Threat score, response policy, mitigated hash | Threat scoring module (v1.3+) |

Keys for all eight entity circuits are generated by `setup_entity_zk` / `setup_first_run`. Rust unit tests cover every circuit; the Python bridge only invokes the **attest-path** subset above.

---

## Typical attest bundle (text)

```
Governance: redaction, rule-binding, pipeline
Entity:     redaction-v1, core-redaction, batch-merkle (if 1–4 entities),
            merkle-inclusion×N (one per entity, max 8), compliance (if redaction occurred)
```

## Typical attest bundle (voice)

```
Governance: (same as text)
Entity:     redaction-v1, core-redaction, batch-merkle (if applicable), voice-redaction
```

`verify_attestation --real-zk` verifies every proof present in the artifact's `entity_proofs.proofs` map.
